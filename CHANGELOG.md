All notable changes to this project will be documented in this file.

## [0.5.0-candidate] - 2026-06-28

### Added

* Added `Collective Agent Rhythm Hook` as the fifth schema of the Collective Pranayama Protocol.
* Added `schemas/collective-agent-rhythm-hook.schema.json`.
* Added `examples/collective-agent-rhythm-hook.example.yaml`.
* Updated `scripts/validate_examples.py` to validate:

  * `Collective Rhythm Record`
  * `Collective Synchronization Policy`
  * `Collective Rhythm Route`
  * `Collective Exhalation Bridge`
  * `Collective Agent Rhythm Hook`

### Changed

* Extended the protocol from collective exhalation handoff to AI-agent operability.
* Added support for:

  * agent group identity and autonomy level;
  * collective rhythm signal detection;
  * synchronization policy invocation;
  * rhythm route invocation;
  * collective exhalation bridge invocation;
  * automatic execution boundaries;
  * human review requirements;
  * traceability chain links;
  * expected pressure, redundancy, diversity, and coordination outcomes.

### Design Principle

* Established the v0.5 principle:

  > Agents may synchronize, but never silence diversity.

### Positioning

* `v0.1` records collective rhythm events.
* `v0.2` defines when and how collective synchronization should be allowed.
* `v0.3` defines how synchronized rhythm is routed across participants.
* `v0.4` defines how collective computational exhaust is safely handed off to the Exhalation Layer.
* `v0.5` defines how AI agent groups safely invoke the collective rhythm flow.

This completes the first arc of the Collective Pranayama Protocol and prepares it for future agent swarm workflows, distributed memory hygiene, shared trace reuse, and collective computational metabolism.
