# Changelog

All notable changes to this project will be documented in this file.

[0.2.0-candidate] - 2026-06-28
Added
Added Collective Synchronization Policy as the second schema of the Collective Pranayama Protocol.
Added schemas/collective-synchronization-policy.schema.json.
Added examples/collective-synchronization-policy.example.yaml.
Updated scripts/validate_examples.py to validate both:
Collective Rhythm Record
Collective Synchronization Policy
Changed
Extended the protocol from collective rhythm recording to synchronization policy.
Added support for:
synchronization scope definition;
rhythm triggers;
coordination rules;
synchronization intensity limits;
output diversity boundaries;
shared exhalation boundaries;
human review boundaries;
pressure, redundancy, and diversity expectations.
Design Principle

Established the v0.2 principle:

Synchronize by policy, not by panic.

Positioning
v0.1 records collective rhythm events.
v0.2 defines when and how collective synchronization should be allowed.

This makes the protocol safer for future distributed agent workflows, shared trace reuse, collective exhalation, and memory pressure balancing.

* Added `schemas/collective-rhythm-record.schema.json`.
* Added `examples/collective-rhythm-record.example.yaml`.
* Added validation script at `scripts/validate_examples.py`.
